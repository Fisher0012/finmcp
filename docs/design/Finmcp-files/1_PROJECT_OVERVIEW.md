# FinMCP — 项目总览

> 给 AI 财经/投研创作者的开源 MCP 工具集
> Open-Source MCP Toolkit for Financial Content Creators & AI Developers

---

## 1. 一句话定位

**FinMCP 是一组面向 A 股市场和财经内容创作场景的 MCP（Model Context Protocol）工具集，让 Claude、Cursor、Claude Code 等支持 MCP 协议的 AI 客户端，能直接获取 A 股真实数据、生成专业财经分析、产出符合中文社媒规范的内容。**

## 2. 项目背景

### 2.1 问题陈述

当前 AI 助手（Claude、ChatGPT 等）在处理中国金融/财经场景时存在三个核心缺陷：

1. **数据缺失**：无法实时获取 A 股行情、财务数据、宏观经济指标
2. **数据幻觉**：基于训练语料"编造"股票数据，错误率高，金融场景容错为零
3. **本土化不足**：海外工具不理解 A 股特殊机制（涨跌停、ST、停牌、龙虎榜等）

### 2.2 市场机会窗口

- **MCP 生态爆发期**：Anthropic 在 2024 年 11 月正式推出 MCP 协议，2025-2026 是开发者生态建设关键期
- **中文 MCP 服务严重稀缺**：截至 2026 年 5 月，MCP 官方注册表中针对中国金融市场的高质量 server 几乎为零
- **目标用户群快速增长**：使用 Claude / Cursor / Claude Code 的中文 AI 开发者 + 财经创作者群体正在快速扩张

### 2.3 创始人优势

- 已有完整 A 股量化数据基础设施（22M 行 10 年历史数据，SQLite + Python）
- 已运行多个生产级数据系统（每日估值推送、量化策略 B、xhs-auto-reply 等）
- 财经内容创作者身份（Xhs 120k 粉、公众号"阿信财读笔记"），既是开发者又是目标用户
- 全职 AI 开发投入

## 3. 产品形态

### 3.1 不是什么

- 不是 SaaS 平台（不做云端服务、不存储用户数据）
- 不是封闭产品（核心开源，社区驱动）
- 不是 GUI 应用（无前端界面，纯协议层）

### 3.2 是什么

- **一组独立的 MCP server**，每个解决一个具体问题
- **本地部署优先**：用户通过 `pip install` 或 `uvx` 在本地运行
- **协议合规**：完全遵循 MCP 标准，可被任何 MCP 客户端调用
- **AI-Native**：工具接口为 LLM 调用而设计，输出结构对 LLM 友好

## 4. 商业模式

### 4.1 开源核心 + 付费高级版

| 层级 | 内容 | 模式 |
|------|------|------|
| **开源层** | 基础数据获取（akshare 接入）、基础分析工具 | MIT 协议，免费 |
| **付费层** | 高级数据源（tushare Pro / Wind 接入）、专业分析模板、批量处理工具 | 订阅制（¥99-299/月）或一次性买断 |
| **企业层** | 私有部署、定制数据源、私募/券商场景适配 | 定制化报价 |

### 4.2 12 个月收入目标

- **保守目标**：付费用户 200 人，月收入 ¥2 万，年化 ARR ¥25 万
- **乐观目标**：付费用户 500 人 + 2 家企业客户，年化 ARR ¥80 万

收入不是 v1.0 的核心 KPI。**v1.0 的核心 KPI 是 GitHub Stars + 真实周活用户**。

## 5. 长期愿景

FinMCP 是**方向 C（Alpha Press 财经内容工作台）的底层能力层**。

未来 12-18 个月演进路径：

```
v1.0 (M1-M3)  ────► 单个 MCP server（a-stock-data）
                    GitHub 开源 + 个人品牌建设

v1.5 (M4-M6)  ────► 工具集扩展（5-8 个 server）
                    付费版上线 + 第一批付费用户

v2.0 (M7-M12) ────► 与 Alpha Press 产品打通
                    MCP 作为 Alpha Press 的开放接口层
                    形成"产品 + 工具集 + 内容"三位一体生态
```

## 6. 工具集 Roadmap

按优先级排序的 8 个 MCP server：

| 优先级 | 工具名 | 功能 | 状态 | 上线时间 |
|--------|--------|------|------|----------|
| **P0** | `finmcp-a-stock-data` | A 股行情/财务/基础数据 | MVP 开发中 | M1-M2 |
| **P1** | `finmcp-stock-valuation` | 估值计算（PE/PB/PEG/DCF） | 规划中 | M3 |
| **P1** | `finmcp-financial-research` | 财报分析 + 产业链研究 | 规划中 | M4 |
| **P2** | `finmcp-xhs-formatter` | Xhs 财经图文排版 | 规划中 | M5 |
| **P2** | `finmcp-wechat-publisher` | 公众号草稿箱推送 | 规划中 | M6 |
| **P3** | `finmcp-macro-data` | 宏观经济数据（央行、统计局） | 规划中 | M7 |
| **P3** | `finmcp-research-fetch` | 卖方研报抓取与摘要 | 规划中 | M8 |
| **P3** | `finmcp-portfolio-tracker` | 组合追踪与归因 | 规划中 | M9+ |

**MVP 阶段（接下来 3 个月）只做 P0：`finmcp-a-stock-data`，做到极致再扩。**

## 8. 风险与对冲

| 风险 | 概率 | 影响 | 对冲策略 |
|------|------|------|----------|
| MCP 协议本身演进/变化 | 中 | 高 | 紧跟 Anthropic 官方更新，使用 FastMCP 等成熟框架 |
| Anthropic 或第三方推出竞品 | 中 | 中 | 用垂类深度（A 股 + 财经创作）建立壁垒 |
| akshare/tushare 数据合规性 | 低 | 高 | 个人开源使用合规；商业化前接入 tushare Pro / 自建合规数据源 |
| 创作者付费意愿低 | 中 | 中 | 开源核心免费，付费部分聚焦专业用户而非大众 |
| 个人精力分散 | 中 | 高 | 严格按 Roadmap，MVP 阶段砍掉所有 P1+ 功能 |

## 9. 成功的定义

**v1.0 算成功的硬指标**（3 个月内）：

- GitHub Stars ≥ 500
- PyPI 月下载量 ≥ 1000
- 真实活跃用户 ≥ 50（issue/discussion 有交互）
- 至少 2 篇技术社区文章被推荐（V2EX、即刻、Twitter）

**v1.0 算失败的硬指标**：

- 3 个月内 GitHub Stars < 100
- 没有真实用户反馈
- 仅自己使用，社区零互动

失败不等于放弃——失败的话回到方向 A（财经 Agent）或专注 Alpha Press 主产品。
